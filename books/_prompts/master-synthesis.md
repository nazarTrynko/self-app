## Synthesis Validation Checklist

After synthesis, validate with these questions:

### Oracle Validation (Strategic Coherence)
- [ ] Does the synthesis create value beyond individual sources?
- [ ] Is the unique angle clear and defensible?
- [ ] Does the transformation promise resonate with the audience?
- [ ] Are there hidden opportunities not yet explored?

### Critic Validation (Logical Coherence)
- [ ] Do the three portals cover distinct but complementary approaches?
- [ ] Is there logical progression in the chapter structure?
- [ ] Do the exercises build on each other?
- [ ] Are claims supported by source materials?

### Creator Validation (Innovation)
- [ ] Is the core framework genuinely novel?
- [ ] Does the central metaphor create new understanding?
- [ ] Are the reflection tools memorable and shareable?
- [ ] Will users tell others about this?

### Architect Validation (Structure)
- [ ] Is the book structure clear and balanced?
- [ ] Can each section stand alone while supporting the whole?
- [ ] Is the landing page flow logical?
- [ ] Are analytics touchpoints defined?

### Guardian Validation (Integrity)
- [ ] Are sources properly acknowledged?
- [ ] Are claims within evidence bounds?
- [ ] Are potential misinterpretations addressed?
- [ ] Is the pricing fair for value delivered?

## Advanced: Iterative Synthesis

For complex multi-source syntheses:

### Phase 1: Source Analysis (Per Source)
```markdown
Analyze [Source Name]:
1. Core concepts (5-7)
2. Key techniques (3-5)
3. Theoretical foundation
4. Limitations/blind spots
5. Best applications
```

### Phase 2: Connection Mapping
```markdown
Map connections between sources:
1. Overlapping concepts (what they share)
2. Complementary gaps (what one has that others lack)
3. Potential conflicts (where they disagree)
4. Synthesis opportunities (1+1=3 combinations)
```

### Phase 3: Framework Generation
```markdown
Generate unified framework:
1. Name that captures the synthesis
2. Visual model integrating source concepts
3. Process that uses techniques from all sources
4. Unique contribution beyond sources
```

### Phase 4: Validation & Refinement
```markdown
Validate synthesis:
1. Test with target audience description
2. Identify edge cases
3. Refine until coherent
4. Document limitations
```

## Quick Reference: Synthesis Patterns

| Pattern | Description | When to Use |
|---------|-------------|-------------|
| **Layered** | Source A provides foundation, B adds layer, C adds layer | Building comprehensive system |
| **Dialectic** | Source A and B seem opposed; synthesis resolves tension | Reconciling different approaches |
| **Complementary** | Sources address different aspects of same problem | Covering full territory |
| **Enhancement** | One source improves/extends another | When one source is primary |
| **Meta** | Create framework that explains when to use each source | When sources serve different contexts |
